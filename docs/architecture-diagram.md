# /architecture 도식 업데이트 지침

`/architecture` 페이지의 Mermaid 도식은 **수동 관리**한다. 자동 생성·CI 갱신이 없으므로,
구조적 변화가 쌓였다 싶을 때 이 지침대로 한 번씩 갱신한다.

- **SSOT**: `src/content/site-architecture.mermaid.ts` (템플릿 리터럴 안의 Mermaid 한 장)
- **렌더러**: `MermaidDiagram` 컴포넌트 → `/architecture` 페이지 블록으로 노출

## 무엇을 목표로 하는가

이 도식의 독자는 사이트 방문자(특히 builder·curious 페르소나)다. 목표는 코드 지도가 아니라
**"콘텐츠가 어디서 태어나 어떤 경로로 방문자에게 닿는가"를 한 장으로 보여주는 것**이다.

- 다루는 것: 저장소·데이터 스토어·파이프라인(CI/ETL)·외부 서비스·런타임(서버/클라이언트)·방문자, 그리고 그들 사이의 **데이터 흐름**.
- 다루지 않는 것: 개별 컴포넌트/파일, 버그픽스, 카피·스타일 변경, 일회성 스크립트. 노드는 파일이 아니라 **역할** 단위로 잡는다(예: "데이터 로더 3단 폴백" O, `site-data.ts` X).

## 언제 업데이트하는가 (트리거)

아래 유형의 변화가 생겼거나 쌓였을 때. 하나라도 해당하면 갱신 가치가 있다.

1. 새 페이지·서브도메인·라우트 계열 추가 (예: 뉴스룸, /resume)
2. 데이터 스토어·외부 서비스 추가/교체 (예: DB 테이블 계열 신설, LLM 경로 변경)
3. 콘텐츠 파이프라인(CI·ETL·동기화) 단계 변경
4. Ask Hansol 등 API 표면의 구조적 확장 (엔드포인트 추가·역할 분화)
5. 데이터 흐름 방향이 바뀌는 리팩터링

## 갱신 절차

1. **변경 조사** — 도식 파일의 마지막 커밋 이후 구조적 변화를 훑는다.
   ```bash
   LAST=$(git log -1 --format=%H -- src/content/site-architecture.mermaid.ts)
   git log --format="%ad %s" --date=short $LAST..HEAD | grep -E "^[0-9-]+ (feat|refactor)" 
   ```
   `feat`/`refactor` 위주로 보되, 위 트리거 유형에 해당하는 것만 후보로 남긴다.
2. **선별** — 후보를 "방문자에게 닿는 데이터 흐름이 바뀌었나?" 기준으로 거른다.
   애매하면 뺀다(도식이 커지는 것 자체가 비용).
3. **수정** — 아래 스타일 규칙을 지키며 Mermaid를 고친다. 기존 서브그래프 골격
   (온톨로지 vault → CI/Blob → 외부 인프라 → 서버 → 클라이언트 → 방문자)은 유지하고,
   새 영역이 필요할 때만 서브그래프를 추가한다.
4. **검증** — dev 서버에서 `/architecture`를 열어 실제 렌더를 확인한다.
   Mermaid는 문법 오류 시 조용히 빈 화면이 되므로 **스크린샷까지 확인**한다
   (Playwright는 devDependency로 설치돼 있다).
5. **커밋** — `docs(architecture): 도식 갱신 — <반영한 변화 요약>` 형식으로 dev 브랜치에 커밋.
   반영하지 않기로 한 변화가 있으면 커밋 본문에 한 줄로 남긴다(다음 갱신 때 재검토용).

## 스타일 규칙

- **팔레트**: 파일 헤더 주석의 청사진 팔레트 hex만 사용. 새 노드는 기존 `classDef`
  (`mermaid-` 접두 필수 — 전역 CSS와의 선택자 충돌 방지, AGENTS.md 참조)에 편입시키고,
  새 classDef는 기존 12종으로 표현이 불가능할 때만 추가.
- **라벨**: 한국어, 역할 중심, 간결하게. 소스 파일 경로·파일명은 넣지 않는다.
- **엣지**: 실선(`-->`)은 데이터/요청 흐름, 점선(`-.->`)은 참조·분석 관계, 굵은 화살(`==>`)은
  SiteData 등 핵심 콘텐츠 흐름 — 기존 용법을 따른다.
- **크기 억제**: 노드가 늘어나면 같은 역할군을 서브그래프로 묶거나 대표 노드로 합친다.
  한 화면에서 읽히지 않는 도식은 실패다.
- **특수문자 주의**: 라벨 안 괄호·따옴표는 Mermaid 파서가 깨질 수 있다. 기존 라벨처럼
  풀어 쓴다(예: "블로그 1008편").
